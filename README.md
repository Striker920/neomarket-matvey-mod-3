# feat(us-mod-03): одобрение товара модератором

## 🎯 Цель

Реализовать endpoint одобрения товара модератором с переходом карточки из статуса `IN_REVIEW` в `MODERATED` и отправкой события `MODERATED` в B2B-сервис для публикации товара в каталоге.

## 🔍 Контекст

Одобренный товар попадает в каталог B2C — покупатели его видят и заказывают. Если Moderation одобрит, а событие до B2B не дойдёт — товар останется невидимым: продавец потеряет выручку, платформа потеряет доверие.

Модератор подтверждает, что карточка корректна. Цепочка `Moderation → B2B → каталог` должна сработать без сбоев.

## ✅ Что реализовано

### Endpoint
`POST /api/v1/tickets/{ticket_id}/approve`

### Проверки (unhappy paths)
1. **404** — карточка не найдена
2. **403** — карточка в статусе `HARD_BLOCKED`
3. **409** — карточка не в статусе `IN_REVIEW`
4. **403** — карточка закреплена за другим модератором (IDOR-защита)
5. **409** — товар без SKU (нельзя одобрить)
6. **409** — продавец редактировал товар во время review (повторный approve запрещён)

### Happy path
- Переход `IN_REVIEW → MODERATED`
- Установка `date_moderation`
- Отправка события `MODERATED` в B2B (fire-and-forget)

### Формат события MODERATED

```json
{
  "idempotency_key": "uuid",
  "product_id": "product-123",
  "event_type": "MODERATED",
  "occurred_at": "2026-06-21T10:00:00Z",
  "moderator_id": "moderator-789",
  "moderator_comment": "Approved"
}
